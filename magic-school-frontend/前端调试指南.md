## 快速开始

### 方案一：使用 Mock API 服务器（推荐）

适合前端开发和 UI 调试，无需等待后端完成。

#### 1. 安装依赖

```bash
cd magic-school-frontend
npm install
```

#### 2. 启动 Mock API 服务器

```bash
# 在项目根目录执行
python scripts/mock_api_server.py
```

#### 3. 启动前端开发服务器

```bash
# 在另一个终端窗口执行
cd magic-school-frontend
npm run dev
```

#### 4. 访问应用

打开浏览器访问：`http://localhost:5173`

**测试账号**:
- 学生账号：`student` / `password123`
- 家长账号：`parent` / `password123`

**优点**: 完全本地化，无需启动额外服务器
**缺点**: 需要修改代码，测试后需要恢复

### 3. 常见调试场景

#### 场景 1：API 请求失败

**问题**: 页面显示错误，API 请求失败

**排查步骤**:
1. 打开 Network 标签，查看请求详情
2. 检查请求 URL 是否正确
3. 检查请求方法（GET/POST）
4. 检查请求头（Authorization）
5. 查看响应状态码和内容

**常见原因**:
- Mock API 服务器未启动
- API URL 配置错误（`.env` 文件）
- Token 已过期
- CORS 跨域问题

**解决方案**:
```bash
# 1. 确认 Mock API 服务器运行中
curl http://localhost:3000/api/v1/health

# 2. 检查环境变量配置
cat magic-school-frontend/.env

# 3. 清除浏览器缓存和 localStorage
# 在浏览器 Console 中执行
localStorage.clear()
location.reload()
```

#### 场景 2：页面样式错乱

**问题**: 样式不正确，布局错位

**排查步骤**:
1. 打开 Elements 标签
2. 检查元素的 computed 样式
3. 查看是否有 CSS 冲突
4. 检查 Tailwind CSS 类名

**解决方案**:
```bash
# 重新构建 Tailwind CSS
cd magic-school-frontend
npm run build
```

#### 场景 3：路由跳转失败

**问题**: 点击导航无响应或跳转错误

**排查步骤**:
1. 检查路由配置（`App.tsx`）
2. 检查 `Link` 组件的 `to` 属性
3. 检查 `PrivateRoute` 权限配置
4. 查看浏览器 Console 错误

#### 场景 4：认证失败

**问题**: 无法登录，Token 无效

**排查步骤**:
1. 打开 Application 标签
2. 查看 Local Storage 中的 Token
3. 检查 Token 格式是否正确
4. 查看登录 API 响应

**解决方案**:
```bash
# 1. 清除认证信息
# 在浏览器 Console 中执行
localStorage.clear()

# 2. 重新登录
```

## Mock API 服务器

### 功能说明

Mock API 服务器提供前端开发所需的所有数据接口：

#### 认证接口
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/register` - 用户注册
- `GET /api/v1/auth/me` - 获取当前用户信息

#### 数据接口
- `GET /api/v1/dashboard/:studentName` - 获取仪表盘数据
- `GET /api/v1/schedule/:studentName` - 获取课程表
- `GET /api/v1/achievements/:studentName` - 获取成就数据
- `GET /api/v1/homework/:studentName` - 获取作业列表
- `GET /api/v1/profile/:studentName` - 获取学生档案
- `GET /api/v1/points/:studentName` - 获取积分趋势
- `GET /api/v1/memory/:userId` - 获取长期记忆
- `GET /api/v1/parent/students` - 家长查看学生列表
- `GET /api/v1/parent/conversations/:studentId` - 家长查看对话记录

### 启动方法

```bash
# 确保已安装 Python 依赖
pip install fastapi uvicorn pydantic

# 启动 Mock API 服务器
python scripts/mock_api_server.py
```

服务器将在 `http://localhost:3000` 启动。

### API 测试

```bash
# 测试健康检查
curl http://localhost:3000/api/v1/health

# 测试登录
curl -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"student","password":"password123"}'

# 测试获取仪表盘
curl http://localhost:3000/api/v1/dashboard/test_student \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 常见问题

### Q1: 前端无法连接到 Mock API

**问题**: 显示 "Network Error" 或连接失败

**解决方案**:
1. 确认 Mock API 服务器正在运行
2. 检查 `.env` 文件中的 API URL 配置
3. 检查端口是否被占用
4. 确认没有防火墙拦截

```bash
# 检查 Mock API 是否运行
curl http://localhost:3000/api/v1/health

# 检查端口占用
lsof -i :3000  # Mac/Linux
netstat -ano | findstr :3000  # Windows
```

### Q2: 登录后立即跳转到登录页

**问题**: 登录成功后又被重定向到登录页

**原因**: Token 无效或未正确存储

**解决方案**:
1. 检查 Local Storage 中是否有 Token
2. 检查 Token 格式是否正确
3. 确认 Mock API 返回的 Token 有效
4. 清除缓存重新登录

```javascript
// 在浏览器 Console 中检查
console.log(localStorage.getItem('magic_school_token'))
```

### Q3: 图表不显示

**问题**: 积分趋势图表不显示

**解决方案**:
1. 检查 ECharts 是否正确加载
2. 检查数据格式是否正确
3. 检查图表容器是否有高度和宽度

```javascript
// 检查 ECharts 是否加载
console.log(window.echarts)
```

### Q4: 样式加载异常

**问题**: Tailwind CSS 样式不生效

**解决方案**:
1. 确认 `tailwind.config.js` 配置正确
2. 检查 `index.css` 中是否导入了 Tailwind
3. 重新构建项目

```bash
cd magic-school-frontend
rm -rf node_modules/.vite
npm run dev
```

### Q5: WebSocket 连接失败

**问题**: 无法建立 WebSocket 连接

**解决方案**:
1. 检查 `.env` 中的 WS_URL 配置
2. 确认后端 WebSocket 服务已启动
3. 检查防火墙设置

## 开发建议

### 1. 使用 TypeScript 严格模式

在 `tsconfig.json` 中启用严格模式：

```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true
  }
}
```

### 2. 使用 ESLint

```bash
npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
```

### 3. 使用 Prettier

```bash
npm install -D prettier eslint-config-prettier eslint-plugin-prettier
```

### 4. 组件开发最佳实践

- 使用 TypeScript 类型定义 Props
- 使用 React Hooks 进行状态管理
- 组件职责单一，保持简洁
- 使用注释说明复杂逻辑
