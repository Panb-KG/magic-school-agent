# 页面空白问题排查指南

## 🔍 问题诊断

页面不报错但完全空白，通常是以下原因之一：

1. JavaScript 错误（浏览器控制台有错误信息）
2. CSS 样式问题（内容不可见）
3. 路由配置问题
4. API 调用失败
5. 组件渲染错误

---

## ✅ 排查步骤

### 步骤 1: 打开浏览器开发者工具

1. 在页面上右键点击
2. 选择「检查」或「Inspect」（或按 F12）
3. 查看「Console」（控制台）标签

**查看是否有错误信息：**
- 红色错误信息
- 黄色警告信息
- 任何异常信息

### 步骤 2: 查看 Network 标签

1. 切换到「Network」（网络）标签
2. 刷新页面（Cmd + R）
3. 查看是否有失败的请求（红色）

**重点检查：**
- `/api/v1/*` 请求是否失败
- 请求状态码是否为 401 或 404
- 是否有 CORS 错误

### 步骤 3: 查看页面 HTML

1. 切换到「Elements」（元素）标签
2. 查看 `<div id="root">` 元素
3. 检查是否有内容被渲染

## 🛠️ 常见问题和解决方案

### 问题 1: 控制台显示 401 Unauthorized

**错误信息**: `401 Unauthorized` 或 `Token expired`

**原因**: 未登录或 Token 过期

**解决方案**:

访问登录页面：
```
http://localhost:5173/login
```

使用测试账号登录：
- 学生: `student` / `password123`
- 家长: `parent` / `password123`

---

### 问题 2: 控制台显示 CORS 错误

**错误信息**: `Access-Control-Allow-Origin` 或 `CORS policy`

**原因**: 后端 CORS 配置问题

**解决方案**:

检查 vite.config.ts 中的代理配置：
```typescript
proxy: {
  '/api': {
    target: 'http://101.126.128.57:3000',
    changeOrigin: true
  }
}
```

---

### 问题 3: 控制台显示 "Cannot find module"

**错误信息**: `Cannot find module '@/xxx'`

**原因**: 路径别名配置问题

**解决方案**:

检查 tsconfig.json 中的路径配置：
```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

---

### 问题 4: 控制台显示 "Navigation guard"

**错误信息**: 导航守卫相关错误

**原因**: 未登录自动跳转失败

**解决方案**:

直接访问登录页：
```
http://localhost:5173/login
```

---

### 问题 5: 页面有内容但不可见

**原因**: CSS 样式问题

**解决方案**:

1. 切换到「Elements」标签
2. 查看元素是否有 `display: none` 或 `visibility: hidden`
3. 检查元素的实际渲染内容

---

### 问题 6: API 请求失败

**Network 标签显示失败请求**

**检查后端服务状态**:

在终端运行：
```bash
# 测试后端是否可访问
curl http://101.126.128.57:5000/docs
curl http://101.126.128.57:3000/docs
```

如果无法访问，需要检查：
1. 后端服务是否运行
2. 网络连接是否正常
3. 防火墙设置

## 🎯 快速解决方案

### 方案 1: 直接访问登录页

```
http://localhost:5173/login
```

### 方案 2: 清除浏览器缓存

**Chrome**:
1. 打开开发者工具（F12）
2. 右键点击刷新按钮
3. 选择「清空缓存并硬性重新加载」

**或者按**: `Cmd + Shift + R` (Mac) / `Ctrl + Shift + R` (Windows)

### 方案 3: 重启开发服务器

在终端按 `Ctrl + C` 停止服务，然后重新运行：

```bash
npx vite
```

### 方案 4: 检查环境变量

确认 `.env.development` 文件存在并正确配置：

```bash
cat .env.development
```

应该显示：
```
VITE_BACKEND_URL=http://101.126.128.57:5000
VITE_API_BASE_URL=http://101.126.128.57:3000/api/v1
VITE_WS_URL=ws://101.126.128.57:8765
```

## 📊 完整检查清单

在浏览器开发者工具中检查：

### Console（控制台）
- [ ] 无红色错误
- [ ] 无黄色警告（或警告不影响功能）

### Network（网络）
- [ ] API 请求状态码为 200
- [ ] 无失败的请求
- [ ] 请求响应正常

### Elements（元素）
- [ ] `<div id="root">` 包含内容
- [ ] 元素可见（非 display: none）
