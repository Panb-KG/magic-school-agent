# 测试页面无法访问的解决方案

## 🔍 问题分析

用户反馈无法打开测试页面 http://101.126.128.57:5173/agent-test.html

**根本原因**: 阿里云ECS服务器的公网IP（101.126.128.57）的5173端口未在安全组中开放，导致无法从外部访问。

**验证结果**:
- ✅ 服务正常运行（内网IP可访问）
- ✅ 内网访问: http://9.128.44.145:5173/agent-test.html 正常
- ❌ 公网访问: http://101.126.128.57:5173/agent-test.html 失败

---

## ✅ 解决方案

### 方案 1: 本地IDE调试 - 使用环境变量（推荐）

如果您在本地IDE调试前端，可以直接配置连接到远程后端：

#### 步骤 1: 配置环境变量

在本地项目目录创建 `.env` 文件：

```bash
cd magic-school-frontend
cat > .env << 'EOF'
VITE_BACKEND_URL=http://101.126.128.57:5000
VITE_API_BASE_URL=http://101.126.128.57:3000/api/v1
VITE_WS_URL=ws://101.126.128.57:8765
EOF
```

#### 步骤 2: 重启本地开发服务器

```bash
# 停止当前服务（Ctrl+C）
# 重新启动
npm run dev
```

#### 步骤 3: 访问本地前端

```
http://localhost:5173
```

#### 步骤 4: 测试Agent连接

登录后，在智能对话页面发送消息测试Agent是否正常工作。

---

### 方案 2: 使用SSH端口转发（最安全）

通过SSH隧道将远程服务端口转发到本地：

#### 步骤 1: 建立SSH隧道

```bash
ssh -L 5000:localhost:5000 \
    -L 3000:localhost:3000 \
    -L 8765:localhost:8765 \
    root@101.126.128.57
```

保持这个SSH连接运行。

#### 步骤 2: 配置本地环境变量

```bash
cd magic-school-frontend
cat > .env << 'EOF'
VITE_BACKEND_URL=http://localhost:5000
VITE_API_BASE_URL=http://localhost:3000/api/v1
VITE_WS_URL=ws://localhost:8765
EOF
```

#### 步骤 3: 重启本地开发服务器

```bash
npm run dev
```

#### 步骤 4: 访问本地前端

```
http://localhost:5173
```

**优势**: 所有服务都通过localhost访问，最安全可靠。

---

### 方案 3: 配置Vite代理

修改 `vite.config.ts` 配置代理：

#### 步骤 1: 编辑 vite.config.ts

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import path from 'path'

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src')
    }
  },
  server: {
    host: '0.0.0.0',
    port: 5173,
    proxy: {
      // Agent服务代理
      '/agent': {
        target: 'http://101.126.128.57:5000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/agent/, '')
      },
      // API后端代理
      '/api': {
        target: 'http://101.126.128.57:3000',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '/api')
      },
      // WebSocket代理
      '/ws': {
        target: 'ws://101.126.128.57:8765',
        ws: true,
        changeOrigin: true
      }
    }
  }
})
```

#### 步骤 2: 修改 chatService.ts

```typescript
// 将原来的
const BACKEND_URL = import.meta.env.VITE_BACKEND_URL || 'http://localhost:5000';

// 改为
const BACKEND_URL = '/agent';  // 使用代理路径
```

#### 步骤 3: 重启开发服务器

```bash
npm run dev
```

### 方案 4: 直接测试API（无需前端）

使用curl命令直接测试Agent服务是否工作：

#### 测试 1: Agent健康检查

```bash
curl http://101.126.128.57:5000/health
```

**预期输出**:
```json
{"status":"ok","message":"Service is running"}
```

#### 测试 2: Agent对话（非流式）

```bash
curl -X POST http://101.126.128.57:5000/run \
  -H "Content-Type: application/json" \
  -d '{
    "query": "你好",
    "session_id": "test_session_123"
  }'
```

**预期输出**: 包含Agent的完整响应JSON

#### 测试 3: 测试CORS配置

```bash
curl -X OPTIONS http://101.126.128.57:5000/run \
  -H "Origin: http://localhost:5173" \
  -H "Access-Control-Request-Method: POST" \
  -H "Access-Control-Request-Headers: Content-Type,Authorization" \
  -v 2>&1 | grep -i "access-control"
```

**预期输出**:
```
< access-control-allow-origin: http://localhost:5173
< access-control-allow-methods: DELETE, GET, HEAD, OPTIONS, PATCH, POST, PUT
< access-control-allow-credentials: true
< access-control-allow-headers: Content-Type,Authorization
```

### 方案 5: 开放阿里云安全组（需要服务器权限）

如果您有阿里云控制台权限，可以开放端口：

#### 步骤 1: 登录阿里云控制台

访问: https://ecs.console.aliyun.com/

#### 步骤 2: 进入安全组配置

1. 找到ECS实例
2. 点击「安全组」
3. 点击「配置规则」

#### 步骤 3: 添加入方向规则

添加以下规则：

| 协议类型 | 端口范围 | 授权对象 | 描述 |
|---------|---------|---------|------|
| TCP | 3000/3000 | 0.0.0.0/0 | API后端 |
| TCP | 5000/5000 | 0.0.0.0/0 | Agent服务 |
| TCP | 5173/5173 | 0.0.0.0/0 | 前端服务 |
| TCP | 8765/8765 | 0.0.0.0/0 | WebSocket |

#### 步骤 4: 验证端口开放

```bash
# 从外部测试
curl -I http://101.126.128.57:5173
```
