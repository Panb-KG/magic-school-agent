# 历史对话模块开发文档

## 📋 文档信息

- **文档版本**: 1.0.0
- **创建日期**: 2024
- **模块名称**: 历史对话模块
- **功能描述**: 管理用户与智能体的对话会话历史，支持自动生成对话标题

---

## 🎯 功能概述

历史对话模块提供了完整的对话管理功能，包括：

1. **创建对话会话**：创建新的对话会话，可关联学生
2. **添加消息**：向对话会话中添加用户或助手消息
3. **查询历史对话**：按时间倒序获取历史对话列表
4. **对话标题生成**：使用 LLM 自动生成对话标题
5. **搜索对话**：根据关键词搜索对话
6. **对话详情查看**：查看对话的所有消息记录
7. **删除对话**：删除对话会话及其所有消息
8. **更新标题**：手动更新对话标题

---

## 🗄️ 数据库设计

### 1. 对话会话表 (conversations)

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | Integer | 对话ID（主键） |
| user_id | String(50) | 用户ID |
| student_id | Integer | 关联的学生ID（可选） |
| title | String(256) | 对话标题（由AI生成） |
| summary | Text | 对话摘要（可选） |
| message_count | Integer | 消息数量 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 最后更新时间 |

### 2. 消息表 (messages)

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | Integer | 消息ID（主键） |
| conversation_id | Integer | 对话ID（外键） |
| role | String(32) | 角色（user/assistant） |
| content | Text | 消息内容 |
| created_at | DateTime | 创建时间 |

### 4. 对话标题生成工具

**文件**: `src/tools/conversation_title_tool.py`

提供以下工具函数：

#### generate_conversation_title
```python
@tool
def generate_conversation_title(conversation_id: int, runtime: ToolRuntime = None) -> str
```
为对话自动生成标题。

**参数**:
- `conversation_id`: 对话ID
- `runtime`: 工具运行时上下文

**工作原理**:
1. 获取对话的前10条消息
2. 使用 LLM 分析对话内容
3. 生成简短的标题（不超过15字）
4. 更新对话标题

**返回**: 生成的标题

---

#### generate_title_from_messages
```python
@tool
def generate_title_from_messages(first_message: str, assistant_response: str = "", runtime: ToolRuntime = None) -> str
```
根据用户的第一条消息和助手回复生成对话标题。

**参数**:
- `first_message`: 用户的第一条消息
- `assistant_response`: 助手的回复（可选）
- `runtime`: 工具运行时上下文

**使用场景**: 在创建对话时立即生成标题

---

#### batch_generate_titles
```python
@tool
def batch_generate_titles(days: int = 7, runtime: ToolRuntime = None) -> str
```
批量为最近的对话生成标题。

**参数**:
- `days`: 最近N天的对话（默认7天）
- `runtime`: 工具运行时上下文

**功能**: 为所有默认标题（格式为"对话 - YYYY-MM-DD HH:MM"）的对话生成新标题

## 🔐 权限控制

### 用户身份验证

所有对话工具都会验证用户身份，确保：
1. 用户只能访问自己的对话
2. 用户只能修改和删除自己的对话
3. 家长可以查看关联学生的对话

### 权限检查逻辑

```python
def _get_user_context(runtime: ToolRuntime) -> tuple:
    """从运行时上下文中获取用户信息"""
    if not runtime:
        return None, None

    ctx = runtime.context if hasattr(runtime, 'context') else None
    if not ctx:
        return None, None

    configurable = ctx.get("configurable") if hasattr(ctx, 'get') else {}
    if not configurable:
        return None, None

    user_id = configurable.get("user_id")
    user_role = configurable.get("user_role")

    return user_id, user_role
```

## 📊 性能优化

### 1. 索引优化

数据库表已创建以下索引：

**conversations 表**:
- `ix_conversations_user`: 加快按用户查询
- `ix_conversations_student`: 加快按学生查询
- `ix_conversations_created`: 加快按时间排序

**messages 表**:
- `ix_messages_conversation`: 加快按对话查询消息
- `ix_messages_role`: 加快按角色筛选
- `ix_messages_created`: 加快按时间排序

### 2. 查询优化

- 使用 `LIMIT` 限制返回数量
- 使用 `ORDER BY` 确保时间倒序
- 使用 `JOIN` 减少查询次数

### 3. 标题生成优化

- 只取前10条消息生成标题
- 限制标题长度（不超过15字）
- 批量生成减少 LLM 调用次数

## 🚀 前端集成指南

### 1. API 端点


### 2. React 组件示例

```tsx
import React, { useState, useEffect } from 'react';

interface Conversation {
  id: number;
  title: string;
  created_at: string;
  message_count: number;
}

function ConversationList() {
  const [conversations, setConversations] = useState<Conversation[]>([]);

  useEffect(() => {
    fetch('/api/conversations')
      .then(res => res.json())
      .then(data => setConversations(data.conversations));
  }, []);

  return (
    <div>
      <h2>历史对话</h2>
      <ul>
        {conversations.map(conv => (
          <li key={conv.id}>
            <h3>{conv.title}</h3>
            <p>{conv.message_count}条消息</p>
            <p>{new Date(conv.created_at).toLocaleString()}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

## 📝 注意事项

### 1. 标题生成

- 标题生成需要调用 LLM，可能需要几秒钟
- 建议在对话创建后异步生成标题
- 如果 LLM 调用失败，使用默认标题

### 2. 消息存储

- 消息内容可能很长，需要考虑存储空间
- 建议定期清理过期的对话记录
- 可以添加消息内容长度限制

### 3. 权限控制

- 始终验证用户身份
- 家长只能查看关联学生的对话
- 学生只能查看自己的对话

### 4. 性能考虑

- 大量对话时建议分页查询
- 定期清理旧对话以减少数据库负载
- 考虑添加对话归档功能
