## 🔧 如果仍然无法访问

### 1. 检查防火墙

```bash
# 查看 iptables 规则
sudo iptables -L -n | grep 5173

# 查看防火墙状态
sudo ufw status
```

### 2. 开放端口

```bash
# 使用 iptables 开放端口
sudo iptables -I INPUT -p tcp --dport 5173 -j ACCEPT

# 或使用 ufw（如果可用）
sudo ufw allow 5173/tcp
```

### 3. 检查云服务安全组

如果你使用的是云服务器（如阿里云、腾讯云），需要在控制台的安全组中开放 5173 端口。

### 4. 使用端口转发

如果 5173 端口被限制，可以使用 Nginx 进行反向代理：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:5173;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 🌐 网络类型说明

### 169.254.x.x 地址
- **类型**: 链路本地地址（Link-Local）
- **用途**: 用于自动地址分配（APIPA）
- **限制**: 通常只能在同一链路内通信
- **适用场景**: 内部网络、容器环境

### 公网地址
- **类型**: 公网 IP
- **用途**: 可从互联网访问
- **优势**: 无访问限制
- **注意**: 需要考虑安全性
