# 前端无法从公网访问 - 完整解决方案

## 🔍 问题诊断

经过测试，发现以下情况：

- ✅ **本地访问正常**: http://localhost:5173 和 http://localhost:80 都可以访问
- ✅ **端口监听正常**: 服务正在监听 0.0.0.0:5173 和 0.0.0.0:80
- ❌ **公网无法访问**: http://101.126.128.57:5173 和 http://101.126.128.57 都无法访问

## 🎯 问题原因

这是 **云服务提供商的防火墙/安全组配置问题**。

服务器上的防火墙规则正常，但云服务商（如阿里云、腾讯云、华为云等）的**安全组**阻止了外部访问。

---

## 🛠️ 解决方案

### 方案 1: 配置云服务商安全组（推荐）

#### 阿里云 ECS
1. 登录阿里云控制台
2. 进入「云服务器 ECS」
3. 找到你的实例，点击「更多」→「网络和安全组」→「安全组配置」
4. 点击「配置规则」→「添加安全组规则」
5. 添加以下规则：
   ```
   协议类型: TCP
   端口范围: 80/80, 443/443, 5173/5173
   授权对象: 0.0.0.0/0
   优先级: 1
   ```

#### 腾讯云 CVM
1. 登录腾讯云控制台
2. 进入「云服务器」
3. 找到你的实例，点击「更多」→「安全组」→「配置安全组」
4. 点击「添加规则」
5. 添加以下规则：
   ```
   类型: 自定义
   来源: 0.0.0.0/0
   协议端口: TCP:80, TCP:443, TCP:5173
   策略: 允许
   ```

#### 华为云 ECS
1. 登录华为云控制台
2. 进入「弹性云服务器」
3. 找到你的实例，点击「更多」→「网络」→「安全组」
4. 点击「更改安全组」或「配置规则」
5. 添加入方向规则：
   ```
   协议: TCP
   端口: 80, 443, 5173
   源地址: 0.0.0.0/0
   动作: 允许
   ```

### 方案 4: 使用内网穿透工具（临时方案）

安装 ngrok 进行内网穿透：

```bash
# 下载 ngrok
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-amd64.zip
unzip ngrok-v3-stable-linux-amd64.zip

# 注册 ngrok 账号获取 authtoken
# ngrok config add-authtoken YOUR_TOKEN

# 启动隧道
./ngrok http 5173
```

ngrok 会给你一个临时的公网 URL，可以直接访问。

### 方案 5: 使用 Nginx 反向代理（生产环境推荐）

如果端口 80 或 443 被允许访问，可以配置 Nginx：

1. 安装 Nginx：
```bash
apt update
apt install nginx -y
```

2. 配置 Nginx：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:5173;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

3. 重启 Nginx：
```bash
systemctl restart nginx
```

然后访问：`http://your-domain.com`
