# 🚀 魔法课桌智能体 - 扣子平台部署指南

## 📋 部署概览

本文档指导您将魔法课桌智能体部署到扣子平台，提供多种部署方案供选择。

---

## 🎯 部署方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **方案一：外部 API 集成** | 保留现有代码、功能完整 | 需要维护服务器 | 已有完善后端 |
| **方案二：扣子工作流** | 无需服务器、易维护 | 需要重新配置 | 快速上线 |
| **方案三：混合模式** | 兼顾性能与成本 | 架构复杂 | 生产环境 |

## 方案一：外部 API 集成（推荐用于生产环境）

### 原理

在扣子平台创建一个 Bot，通过插件或工作流调用您在阿里云 ECS 上部署的 Agent 服务。

### 步骤 1：准备后端服务

#### 1.1 确保服务可公网访问

```bash
# 检查服务状态
/workspace/projects/scripts/manage_agent.sh status

# 应该看到：
# ✅ 服务正在运行
# 端口 5000 正在监听
```

#### 1.2 配置阿里云安全组

1. 登录阿里云 ECS 控制台
2. 找到实例安全组
3. 添加入站规则：
   - **端口范围**: `5000/5000`
   - **授权对象**: `0.0.0.0/0`（或特定 IP）

#### 1.3 测试公网访问

```bash
# 替换 101.126.128.57 为您的服务器 IP
curl http://101.126.128.57:5000/health

# 预期输出：
# {"status":"ok","message":"Service is running"}
```

#### 1.4 配置 CORS（如果需要）

确认 `/workspace/projects/src/main.py` 中已配置 CORS：

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 生产环境建议指定域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 步骤 4：创建自定义插件

#### 4.1 创建插件

1. 在 Bot 编辑页面，点击"插件"
2. 点击"添加插件"
3. 选择"自定义插件"
4. 填写插件信息：
   - **插件名称**: 魔法课桌 Agent
   - **插件描述**: 调用魔法课桌智能体后端服务
   - **插件图标**: 选择合适图标

#### 4.2 配置插件工具

创建工具：`对话智能体`

**工具配置**：

- **工具名称**: 对话智能体
- **工具描述**: 与魔法课桌智能体进行对话，获取学习建议和管理帮助
- **请求方式**: POST
- **请求地址**: `http://101.126.128.57:5000/run`
- **请求头**:
  ```json
  {
    "Content-Type": "application/json"
  }
  ```
- **请求体**:
  ```json
  {
    "message": "{{message}}",
    "session_id": "{{session_id}}",
    "user_id": "{{user_id}}"
  }
  ```
- **响应解析**:
  ```json
  {
    "回复": "{{messages.[?(@.type=='ai')].content.[0]}}"
  }
  ```

**输入参数**：

| 参数名称 | 类型 | 必填 | 描述 |
|---------|------|------|------|
| message | string | 是 | 用户输入的消息 |
| session_id | string | 是 | 会话ID，用于保持对话上下文 |
| user_id | string | 是 | 用户ID |

**输出参数**：

| 参数名称 | 类型 | 描述 |
|---------|------|------|
| 回复 | string | Agent 的回复内容 |

### 步骤 5：创建工作流（高级方式）

如果需要更复杂的逻辑，可以使用工作流：

#### 5.1 创建工作流

1. 在 Bot 编辑页面，点击"工作流"
2. 点击"创建工作流"
3. 填写工作流信息：
   - **工作流名称**: 魔法课桌对话流程
   - **工作流描述**: 处理用户对话，调用后端 Agent

#### 5.2 配置工作流节点

**节点配置**：

1. **开始节点**
   - 输入：`user_message`, `user_id`, `session_id`

2. **HTTP 请求节点**
   - 节点名称：调用 Agent 服务
   - 请求方法：POST
   - 请求地址：`http://101.126.128.57:5000/run`
   - 请求体：
     ```json
     {
       "message": "{{user_message}}",
       "session_id": "{{session_id}}",
       "user_id": "{{user_id}}"
     }
     ```
   - 响应变量：`agent_response`

3. **结束节点**
   - 输出：`{{agent_response.messages.[?(@.type=='ai')].content.[0]}}`

## 方案二：直接在扣子平台构建（推荐用于快速上线）

### 原理

使用扣子平台的可视化界面重新构建 Agent，无需自己部署服务器。

### 步骤 1：创建 Bot

同方案一步骤 2

### 步骤 2：配置 Agent

#### 2.1 选择 Agent 模式

1. 在 Bot 编辑页面，点击"Agent"
2. 选择"对话型 Agent"

#### 2.2 配置大模型

1. 选择豆包模型
   - **模型选择**: 豆包·Lite-4K（免费）或 豆包·Pro-128K

   - **Top P**: 0.95

#### 2.3 编写系统提示词

复制 `config/agent_llm_config.json` 中的 `sp` 字段内容到系统提示词。

## 方案三：混合模式（高级）

### 原理

将核心逻辑部署在扣子平台，复杂功能调用外部 API。

### 架构图

```
扣子 Bot
    ↓
扣子 Agent（处理通用对话）
    ↓
条件判断
    ↓
    ├─ 简单请求 → 扣子 Agent 直接回复
    └─ 复杂请求 → 调用外部 API (阿里云 ECS)
```

### 实现方式

在扣子工作流中添加条件节点：

```yaml
如果消息包含"创建学生":
  调用外部 API
否则:
  使用本地 Agent
```

## 🆘 常见问题

### Q: 如何获取扣子 Bot 的访问链接？

A: Bot 发布后，在扣子平台可以看到 Bot 的访问链接和二维码。

### Q: 如何限制 Bot 的访问权限？

A: 在扣子平台的 Bot 设置中，可以设置为"仅限邀请用户访问"。

### Q: 后端服务宕机了怎么办？

A: 使用 `./scripts/manage_agent.sh restart` 重启服务，或配置自动重启。

### Q: 如何更新 Bot 的功能？

A:
- 方案一：更新后端服务代码并重启
- 方案二：在扣子平台重新配置 Bot
