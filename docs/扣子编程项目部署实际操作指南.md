## 🎯 部署配置步骤

### 步骤 1：确认部署版本

**界面元素**：下拉选择框
- **当前显示**：`chore: 完成项目清理，添加发布文档和验证脚本`（最新）

### 步骤 4：配置生产环境变量（⚠️ 重要）

**界面元素**：折叠面板 `查看详情`

**当前状态**：`暂无数据`

**操作**：
1. 点击 `查看详情` 展开
2. 查看当前配置的环境变量（如果有）
3. 点击 `+ 新建变量` 添加必要的环境变量

#### 4.1 数据库连接（如果使用外部数据库）
- **变量名**: `DATABASE_URL`
- **变量值**: `postgresql://username:password@host:port/database_name`
- **说明**: 如果使用Coze提供的数据库，可以跳过此步骤

#### 4.2 JWT密钥（必须配置）
- **变量名**: `JWT_SECRET`
- **变量值**: `your-secret-key-here-please-change-this`
- **说明**: 用于JWT Token签名，请使用强随机字符串

#### 4.3 阿里云百炼API密钥（必须配置）
- **变量名**: `DASHSCOPE_API_KEY`
- **变量值**: `sk-f02e3eb3ddff4251a1aaddac91ccb724`（替换为您的实际API Key）
- **说明**: 阿里云百炼API的访问密钥

#### 4.4 阿里云百炼API地址（必须配置）
- **变量名**: `OPENAI_BASE_URL`
- **变量值**: `https://dashscope.aliyuncs.com/compatible-mode/v1`
- **说明**: 阿里云百炼API的访问地址（使用OpenAI兼容接口）

#### 4.5 调试模式
- **变量名**: `DEBUG`
- **变量值**: `False`
- **说明**: 生产环境建议设置为False

#### 4.6 日志级别
- **变量名**: `LOG_LEVEL`
- **变量值**: `INFO`
- **说明**: 日志级别，可选：DEBUG, INFO, WARNING, ERROR

**配置示例**：
```
JWT_SECRET = your-super-secret-jwt-key-change-in-production
DASHSCOPE_API_KEY = sk-f02e3eb3ddff4251a1aaddac91ccb724
OPENAI_BASE_URL = https://dashscope.aliyuncs.com/compatible-mode/v1
DEBUG = False
LOG_LEVEL = INFO
```

**重要提示**：
- ⚠️ `DASHSCOPE_API_KEY` 和 `OPENAI_BASE_URL` 是必须配置的，否则AI对话功能无法使用
- ⚠️ 请勿将真实的API Key泄露到公开的代码仓库中
- ⚠️ 生产环境中的JWT_SECRET必须使用强随机字符串
- ⚠️ 环境变量不应以 `COZE_` 开头，应使用标准的命名规范

**环境变量命名说明**：
- ✅ 使用 `DASHSCOPE_API_KEY`（阿里云百炼标准命名）
- ✅ 使用 `OPENAI_BASE_URL`（OpenAI兼容接口标准命名）
- ❌ 不要使用 `COZE_WORKLOAD_IDENTITY_API_KEY`
- ❌ 不要使用 `COZE_INTEGRATION_MODEL_BASE_URL`

### 步骤 5：查看更多配置（不可修改）

**界面元素**：折叠面板 `更多配置`

**当前显示**：
- **部署类型**: 动态应用（固定，不可修改）
- **服务器资源**: CPU: 1 RAM: 2 GiB（固定，不可修改）
- **构建指令**: `bash scripts/setup.sh`（可修改，通常不需要改）
- **运行指令**: `bash scripts/http_run.sh -p 5000`（可修改，通常不需要改）
- **端口配置**: `0.0.0.0:80`（固定，不可修改）

**重要说明**：

#### 端口配置说明

**实际的端口配置如下**：

1. **应用内部端口**: `5000`
   - 这是FastAPI应用实际监听的端口
   - 由运行指令 `bash scripts/http_run.sh -p 5000` 中的 `-p 5000` 参数控制
   - 可以通过修改运行指令来更改（通常不需要）

2. **外部访问端口**: `80`
   - 这是外部访问应用的端口
   - 通过反向代理将外部80端口转发到内部5000端口
   - 固定不可修改

**访问方式**：
- 通过域名访问时，无需指定端口（默认使用80端口）
- 例如：`https://mq95rx99rp.coze.site`（会自动使用80端口）

**无需手动配置端口**：
- ✅ 默认配置已经正确，无需修改
- ✅ 外部访问使用80端口（HTTPS）
- ✅ 内部应用使用5000端口
- ✅ 系统会自动处理端口转发

**其他配置说明**：

- **部署类型**：固定为"动态应用"，无需修改
- **服务器资源**：固定为 CPU: 1 RAM: 2 GiB，无需修改
- **构建指令**：`bash scripts/setup.sh`，用于安装依赖，通常不需要修改
- **运行指令**：`bash scripts/http_run.sh -p 5000`，用于启动应用，通常不需要修改

**建议**：
- 保持默认配置即可
- 除非有特殊需求，否则不要修改这些配置
- 大部分配置都是固定的，只有环境变量需要手动配置

```typescript
// 开发环境

// 生产环境 - 替换为实际的部署域名
const API_BASE_URL = 'https://mq95rx99rp.coze.site';

export const api = axios.create({
  baseURL: API_BASE_URL,
  timeout: 30000,
});
```

## 🔧 常见问题排查

### 问题 1：部署失败，提示"依赖安装失败"

**可能原因**：
- Python版本不兼容
- 网络连接问题
- 依赖包安装超时

**解决方案**：
1. 检查部署日志中的具体错误信息
2. 确认requirements.txt中的依赖版本
3. 尝试重新部署

### 问题 2：部署成功但无法访问

**可能原因**：
- 端口配置问题
- 防火墙阻止
- 服务未正常启动

**解决方案**：
1. 确认使用域名访问（无需指定端口）
2. 查看部署日志中的服务启动信息
3. 尝试访问 `/health` 端点确认服务是否运行

### 问题 3：AI对话功能不工作

**可能原因**：
- 环境变量未配置或配置错误
- API Key无效或过期
- 环境变量名称不正确

**解决方案**：
1. 检查是否正确配置了 `DASHSCOPE_API_KEY`
2. 检查是否正确配置了 `OPENAI_BASE_URL`
3. 确认环境变量名称不包含 `COZE_` 前缀
4. 查看日志中的错误信息
5. 确认API Key是否有效

### 问题 4：数据库连接失败

**可能原因**：
- 数据库连接字符串错误
- 数据库服务未启动
- 权限不足

**解决方案**：
1. 检查数据库连接配置
2. 确认数据库服务是否正常运行
3. 检查数据库用户权限

### 问题 5：WebSocket 连接失败

**可能原因**：
- WebSocket 端点配置错误
- 防火墙阻止WebSocket连接
- 协议不匹配（ws vs wss）

**解决方案**：
1. 确认使用 `wss://` 协议（生产环境）
2. 检查防火墙设置
3. 查看服务器日志

### 问题 6：访问时提示端口错误

**可能原因**：
- 在URL中指定了端口

**解决方案**：
- ✅ 正确：`https://mq95rx99rp.coze.site`
- ❌ 错误：`https://mq95rx99rp.coze.site:8000`
- ❌ 错误：`https://mq95rx99rp.coze.site:5000`

**说明**：
- 外部访问使用80端口（HTTPS），无需在URL中指定
- 系统会自动处理端口转发
