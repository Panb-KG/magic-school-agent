## 🎯 部署配置步骤

### 步骤 1：确认部署版本

**界面元素**：下拉选择框
- **当前显示**：`7e04a06 docs: 更正为部署按钮的操作指南`（7分钟前）
- **操作**：
  1. 点击下拉箭头查看所有可用版本
  2. 选择最新的版本（通常是列表顶部的）
  3. 确认版本信息正确

**注意**：
- 版本格式：`commit_id 提交说明`
- 时间戳显示提交时间
- 选择最新版本以确保部署最新代码

### 步骤 2：配置部署域名

**界面元素**：输入框
- **当前显示**：`mq95rx99rp.coze.site`
- **作用**：部署后访问该项目的域名

**操作**：
- **方案一**：使用默认域名
  - 保持 `mq95rx99rp.coze.site` 不变
  - 部署后可以直接访问

- **方案二**：自定义域名（如果需要）
  - 修改为您想要的子域名
  - 例如：`magic-desk.coze.site`

**建议**：使用默认域名，部署成功后会直接显示。

**界面元素**：标签组

**当前状态**：`暂无数据`

1. 点击 `查看详情` 展开

**建议**：首次部署可以先跳过，部署成功后再根据需要添加。

## 💻 前端集成

部署成功后，您会获得一个域名，例如：`https://mq95rx99rp.coze.site`

在前端项目中配置：

```javascript
// config/agent.js
export const agentConfig = {
  apiUrl: 'https://mq95rx99rp.coze.site', // 部署域名
  apiKey: '', // 如果配置了环境变量，这里可能不需要
  
  async chat(message, sessionId, userId) {
    const response = await fetch(`${this.apiUrl}/run`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        message,
        session_id: sessionId,
        user_id: userId
      })
    });

    return response.json();
  }
};
```

## 🆘 常见问题

### Q1: 部署失败怎么办？

**解决方案**：

1. 查看 `日志` 标签，查看错误信息
2. 常见错误：
   - 依赖安装失败 → 检查 requirements.txt
   - 数据库同步失败 → 检查数据库配置
   - 端口冲突 → 在"更多配置"中修改端口

### Q2: 如何修改已部署的域名？

**解决方案**：

1. 进入部署界面
2. 修改"部署域名"输入框
3. 重新部署

### Q3: 数据库没有同步成功？

**解决方案**：

1. 检查数据库表是否在同步列表中
2. 查看 `日志` 标签，查看同步详情
3. 确认数据库表结构与代码匹配

### Q4: 部署后访问超时？

**解决方案**：

1. 检查服务是否正常启动
2. 查看部署日志
3. 点击"重新部署"
