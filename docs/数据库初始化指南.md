# 数据库初始化指南

## 概述

魔法课桌学习助手使用 PostgreSQL 数据库存储多用户数据、业务数据和长期记忆。本文档介绍如何初始化数据库和相关表结构。

## 数据库架构

数据库包含 3 个 schema：

### 1. auth (认证和权限管理)
- `users` - 用户表（学生和家长）
- `parent_student_mapping` - 家长-学生关联表
- `permissions` - 权限定义表
- `user_sessions` - 用户会话表

### 2. memory (长期记忆存储)
- `user_profile` - 用户画像
- `conversation_summary` - 对话摘要
- `knowledge_mastery` - 知识掌握度
- `behavior_preferences` - 行为偏好
- `important_conversations` - 重要对话

### 3. public (业务数据)
- `students` - 学生信息表
- `courses` - 课程表（学校和课外）
- `homeworks` - 作业任务表
- `coursewares` - 课件表
- `exercises` - 运动记录表
- `achievements` - 成就表

## 验证初始化

### 检查表是否创建成功

```python
from storage.database.db import get_engine
from sqlalchemy import text

engine = get_engine()
with engine.connect() as conn:
    result = conn.execute(text("""
        SELECT schemaname, tablename
        FROM pg_tables
        WHERE schemaname IN ('public', 'auth', 'memory')
        ORDER BY schemaname, tablename;
    """))
    for row in result.fetchall():
        print(f"{row[0]:10} {row[1]}")
```

### 检查索引

```python
from storage.database.db import get_engine
from sqlalchemy import text

engine = get_engine()
with engine.connect() as conn:
    result = conn.execute(text("""
        SELECT schemaname, tablename, indexname
        FROM pg_indexes
        WHERE schemaname IN ('public', 'auth', 'memory')
        ORDER BY schemaname, tablename, indexname;
    """))
    for row in result.fetchall():
        print(f"{row[0]:10} {row[1]:20} {row[2]}")
```

## 常见问题

### 1. 数据库连接失败

**错误信息**: `ValueError: PGDATABASE_URL is not set`

**解决方案**:
- 检查环境变量 `PGDATABASE_URL` 是否设置
- 如果使用 Coze 集成，确保配置了正确的数据库连接信息

### 2. 表已存在错误

**错误信息**: `relation "xxx" already exists`

**原因**: 脚本使用 `CREATE TABLE IF NOT EXISTS`，但外键或约束可能冲突

**解决方案**:
- 删除现有表重新创建
```sql
DROP SCHEMA IF EXISTS auth CASCADE;
DROP SCHEMA IF EXISTS memory CASCADE;
DROP SCHEMA IF EXISTS public CASCADE;
```
- 然后重新运行初始化脚本

### 3. 权限错误

**错误信息**: `permission denied for schema xxx`

**解决方案**:
- 确保数据库用户有足够的权限（CREATE, ALTER, etc.）
```sql
GRANT ALL PRIVILEGES ON DATABASE your_database TO your_user;
GRANT ALL ON SCHEMA public TO your_user;
GRANT ALL ON SCHEMA auth TO your_user;
GRANT ALL ON SCHEMA memory TO your_user;
```

## 数据模型一致性

### Python 模型与 SQL 对应

项目使用 SQLAlchemy ORM 定义数据模型，确保与 SQL 脚本保持一致：

- `src/storage/database/shared/model.py` - 数据模型定义
- `scripts/init_business_tables.sql` - 表结构定义
- `src/storage/database/*_manager.py` - 数据访问层

**重要**: 修改 SQL 脚本时，必须同步更新 Python 模型定义。

## 维护建议

### 1. 定期备份

```bash
# 备份整个数据库
pg_dump -h localhost -U username -d database_name > backup.sql

# 只备份业务表
pg_dump -h localhost -U username -d database_name -t students -t courses -t homeworks -t coursewares -t exercises -t achievements > business_backup.sql
```

### 2. 监控表大小

```sql
SELECT 
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname IN ('public', 'auth', 'memory')
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;
```

### 3. 索引维护

```sql
-- 重建索引
REINDEX INDEX index_name;

-- 分析表以更新统计信息
ANALYZE table_name;
```
