## 🏗️ 服务概述

魔法课桌智能体包含多个后端服务：

| 服务名称 | 端口 | 用途 | 必需 |
|---------|------|------|------|
| 智能体 API 服务器 | 5000 | 核心智能体对话服务 | ✅ 必需 |
| Mock API 服务器 | 3000 | 前端开发调试数据 | ⚠️ 可选 |
| WebSocket 服务器 | 8765 | 实时消息推送 | ⚠️ 可选 |
| 多用户 API 服务器 | 待定 | 用户认证和管理 | ⚠️ 可选 |

## 🚀 启动方式

### 方式 1: 一键启动脚本（推荐）

```bash
cd /workspace/projects
bash scripts/http_run.sh -p 5000
```

**参数说明**:
- `-p 5000`: 指定端口号为 5000

### 方式 2: 直接运行 Python

```bash
cd /workspace/projects
python src/main.py -m http -p 5000
```

### 方式 3: 使用 uvicorn（生产环境）

```bash
cd /workspace/projects
uvicorn src.main:app --host 0.0.0.0 --port 5000 --reload
```

## 📝 启动步骤

### 完整启动（所有服务）

#### 1. 启动智能体 API 服务器（必需）

```bash
cd /workspace/projects

# 前台启动（可以看到日志）
bash scripts/http_run.sh -p 5000

# 后台启动（推荐）
nohup bash scripts/http_run.sh -p 5000 > logs/agent_api.log 2>&1 &
```

#### 2. 启动 Mock API 服务器（可选）

```bash
# 前台启动
python3 scripts/mock_api_server.py

# 后台启动
nohup python3 scripts/mock_api_server.py > logs/mock_api.log 2>&1 &
```

#### 3. 启动 WebSocket 服务器（可选）

```bash
# 前台启动
python3 src/websocket_server.py

# 后台启动
nohup python3 src/websocket_server.py > logs/websocket.log 2>&1 &
```

### 最小化启动（仅智能体 API）

```bash
cd /workspace/projects
bash scripts/http_run.sh -p 5000
```

这样就可以使用核心的智能体对话功能了。

## 🔧 配置选项

### 修改端口

```bash
# 修改智能体 API 端口为 8000
bash scripts/http_run.sh -p 8000
```

### 修改监听地址

编辑 `src/main.py`，修改 uvicorn 配置：

```python
# 添加 host 参数
if __name__ == "__main__":
    uvicorn.run(
        app, 
        host="0.0.0.0",  # 允许外部访问
        port=5000
    )
```

### 环境变量

创建 `.env` 文件：

```env
# 日志级别
LOG_LEVEL=INFO

# 数据库连接
DATABASE_URL=postgresql://...

# 对象存储
S3_ENDPOINT_URL=...

# API 密钥
API_KEY=your_api_key
```

## 🛠️ 管理命令

### 停止服务

```bash
# 停止所有 Python 服务
pkill -f "python.*main.py"
pkill -f "python.*mock_api"
pkill -f "python.*websocket"

# 停止特定端口的服务
lsof -ti:5000 | xargs kill -9
lsof -ti:3000 | xargs kill -9
```

### 重启服务

```bash
# 停止并重新启动
pkill -f "python.*main.py"
sleep 2
bash scripts/http_run.sh -p 5000
```

### 查看服务状态

```bash
# 查看所有进程
ps aux | grep python

# 查看服务详情
systemctl status your-service-name  # 如果使用 systemd
```

## 🐛 常见问题

### Q1: 端口已被占用

**错误信息**:
```
OSError: [Errno 48] Address already in use
```

**解决方法**:

```bash
# 查找占用端口的进程
lsof -i:5000

# 停止占用端口的进程
kill -9 <PID>

# 或使用其他端口
bash scripts/http_run.sh -p 5001
```

### Q2: 模块导入错误

**错误信息**:
```
ModuleNotFoundError: No module named 'xxx'
```

**解决方法**:

```bash
# 安装依赖
pip install -r requirements.txt

# 或安装特定模块
pip install langchain langgraph fastapi
```

### Q3: 数据库连接失败

**错误信息**:
```
ConnectionError: Could not connect to database
```

**解决方法**:

```bash
# 检查数据库服务是否运行
ps aux | grep postgres

# 检查数据库配置
cat .env | grep DATABASE_URL

# 测试数据库连接
psql $DATABASE_URL
```

### Q4: 权限错误

**错误信息**:
```
PermissionError: [Errno 13] Permission denied
```

**解决方法**:

```bash
# 添加执行权限
chmod +x scripts/http_run.sh

# 使用 sudo 运行
sudo bash scripts/http_run.sh -p 5000
```

### Q5: 服务启动后立即退出

**错误信息**:
```
Process finished with exit code 1
```

**解决方法**:

```bash
# 查看详细错误
python src/main.py -m http -p 5000

# 检查日志
cat /app/work/logs/bypass/app.log

# 检查配置文件
cat config/agent_llm_config.json
```

## 📊 性能优化

```bash
# 使用 gunicorn（推荐）

  --workers 4 \
```

```bash
# 限制 CPU 使用
taskset -c 0,1 python src/main.py -m http -p 5000
```

## 🚦 健康检查

### 创建健康检查脚本

```bash
#!/bin/bash
# scripts/health_check.sh

check_service() {
    local name=$1
    local url=$2
    local port=$3
    
    if curl -s -f "$url" > /dev/null 2>&1; then
        echo "✅ $name: 运行正常 (端口 $port)"
    else
        echo "❌ $name: 未运行 (端口 $port)"
        return 1
    fi
}

echo "=== 服务健康检查 ==="
check_service "智能体 API" "http://localhost:5000/docs" 5000
check_service "Mock API" "http://localhost:3000/docs" 3000
check_service "WebSocket" "http://localhost:8765" 8765
```

运行健康检查：

```bash
chmod +x scripts/health_check.sh
bash scripts/health_check.sh
```

```bash

    compress
    delaycompress
    missingok
    notifempty
```
