## 📋 部署清单

### 步骤 1: 开放安全组端口（必须）

#### 1.1 登录阿里云控制台
```
https://ecs.console.aliyun.com/
```

#### 1.2 进入安全组配置
1. 找到您的ECS实例
2. 点击「安全组」标签
3. 点击「配置规则」

#### 1.3 添加入方向规则

**点击「手动添加」**，依次添加以下规则：

| 协议类型 | 端口范围 | 授权对象 | 描述 |
|---------|---------|---------|------|
| 自定义TCP | 80/80 | 0.0.0.0/0 | HTTP |
| 自定义TCP | 443/443 | 0.0.0.0/0 | HTTPS |
| 自定义TCP | 3000/3000 | 0.0.0.0/0 | API后端 |
| 自定义TCP | 5000/5000 | 0.0.0.0/0 | Agent服务 |
| 自定义TCP | 5173/5173 | 0.0.0.0/0 | 前端服务 |
| 自定义TCP | 8765/8765 | 0.0.0.0/0 | WebSocket |

**点击「保存」**

---

### 步骤 2: 配置Nginx反向代理（推荐）

#### 2.1 安装Nginx

```bash
# 在ECS服务器上执行
sudo apt update
sudo apt install nginx -y
```

#### 2.2 配置Nginx

```bash
# 创建配置文件
sudo nano /etc/nginx/sites-available/magic-school
```

#### 2.3 添加配置内容

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为您的域名

    # 前端服务
    location / {
        proxy_pass http://localhost:5173;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # API后端
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Agent服务
    location /agent {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket
    location /ws {
        proxy_pass http://localhost:8765;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

#### 2.4 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/magic-school /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 步骤 3: 配置SSL证书（HTTPS）

#### 3.1 安装Certbot

```bash
sudo apt install certbot python3-certbot-nginx -y
```

#### 3.2 申请SSL证书

```bash
sudo certbot --nginx -d your-domain.com
```

按照提示输入邮箱和同意条款。

#### 3.3 自动续期

```bash
sudo certbot renew --dry-run
```

---

### 步骤 4: 配置域名DNS（如果使用域名）

#### 4.1 在阿里云DNS控制台

1. 进入「云解析DNS」
2. 找到您的域名
3. 添加A记录：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | @ | 您的服务器IP |
| A | www | 您的服务器IP |

#### 4.2 等待DNS生效

通常需要10-30分钟。

## 🔧 可选配置

### 配置1: 使用PM2管理进程

#### 安装PM2

```bash
npm install -g pm2
```

#### 创建启动脚本

```bash
# 创建 ecosystem.config.js
cd /workspace/projects
cat > ecosystem.config.js << 'EOF'
module.exports = {
  apps: [
    {
      name: 'magic-school-frontend',
      script: 'npx',
      args: 'vite',
      cwd: '/workspace/projects/magic-school-frontend',
      instances: 1,
      exec_mode: 'fork',
      watch: false,
      env: {
        NODE_ENV: 'production'
      }
    },
    {
      name: 'magic-school-api',
      script: 'python',
      args: 'scripts/mock_api_server.py',
      cwd: '/workspace/projects',
      instances: 1,
      exec_mode: 'fork'
    },
    {
      name: 'magic-school-agent',
      script: 'python',
      args: 'src/main.py -m http -p 5000',
      cwd: '/workspace/projects',
      instances: 1,
      exec_mode: 'fork'
    }
  ]
}
EOF
```

#### 启动服务

```bash
pm2 start ecosystem.config.js
pm2 save
pm2 startup
```

### 配置2: 配置自动备份

#### 创建备份脚本

```bash
cat > /workspace/projects/scripts/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/backup/magic-school"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

# 备份数据库
pg_dump -U your_user -h localhost magic_school > $BACKUP_DIR/db_$DATE.sql

# 备份配置文件
tar -czf $BACKUP_DIR/config_$DATE.tar.gz /workspace/projects/config

# 清理7天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete

echo "Backup completed: $DATE"
EOF

chmod +x /workspace/projects/scripts/backup.sh
```

#### 配置定时任务

```bash
# 编辑crontab
crontab -e

# 添加每天凌晨2点备份
0 2 * * * /workspace/projects/scripts/backup.sh >> /var/log/backup.log 2>&1
```

## 🆘 常见问题

### Q1: 端口无法访问
**A**: 检查安全组规则是否配置正确

```bash
# 检查端口监听
netstat -tlnp | grep -E "5173|5000|3000|8765"
```

### Q2: HTTPS无法访问
**A**: 检查SSL证书和Nginx配置

```bash
# 检查Nginx配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx
```

### Q3: 服务无法启动
**A**: 查看服务日志

```bash
# 查看Agent日志
tail -f /workspace/projects/logs/agent_service.log

# 查看应用日志
tail -f /app/work/logs/bypass/app.log
```
