## 🚀 快速启动

### 方式 1: 使用一键启动脚本（推荐）

**Linux / macOS:**
```bash
./scripts/start_frontend_dev.sh
```

**Windows:**
```bash
scripts\start_frontend_dev.bat
```

### 方式 2: 手动启动

**步骤 1: 启动 Mock API 服务器**
```bash
python scripts/mock_api_server.py
```

**步骤 2: 启动前端开发服务器**
```bash
cd magic-school-frontend
npm install  # 首次运行
npm run dev
```

**步骤 3: 访问应用**
```
http://localhost:5173
```

## 📝 API 接口列表

### 认证接口
- `POST /api/v1/auth/login` - 用户登录
- `POST /api/v1/auth/register` - 用户注册
- `GET /api/v1/auth/me` - 获取当前用户信息

### 数据接口
- `GET /api/v1/dashboard/:studentName` - 获取仪表盘数据
- `GET /api/v1/schedule/:studentName` - 获取课程表
- `GET /api/v1/achievements/:studentName` - 获取成就数据
- `GET /api/v1/homework/:studentName` - 获取作业列表
- `GET /api/v1/profile/:studentName` - 获取学生档案
- `GET /api/v1/points/:studentName` - 获取积分趋势
- `GET /api/v1/memory/:userId` - 获取长期记忆
- `GET /api/v1/parent/students` - 家长获取学生列表
- `GET /api/v1/parent/conversations/:studentId` - 家长获取对话记录

### 健康检查
- `GET /api/v1/health` - 服务器健康状态

## 🔧 调试技巧

### 1. 查看网络请求

打开浏览器开发者工具（F12）→ Network 标签，查看所有 API 请求。

### 2. 查看控制台日志

打开浏览器开发者工具（F12）→ Console 标签，查看错误和警告。

### 3. 修改 API 地址

编辑 `magic-school-frontend/.env`:
```bash
VITE_API_BASE_URL=http://localhost:3000/api/v1
VITE_WS_URL=ws://localhost:8765
```

### 4. 清除缓存

在浏览器 Console 中执行：
```javascript
localStorage.clear()
location.reload()
```

## ⚠️ 已知问题

### 1. WebSocket 未实现
Mock API 服务器目前不支持 WebSocket，实时通信功能无法测试。

**解决方案**:
- 前端可以正常显示静态数据
- 实时更新功能暂时禁用

### 2. 数据是 Mock 数据
所有数据都是模拟数据，不会持久化。

**解决方案**:
- 可以修改 `scripts/mock_api_server.py` 中的模拟数据
- 后续可以对接真实后端 API

### 3. 部分 npm 包有安全警告

```
31 packages are looking for funding
2 moderate severity vulnerabilities
```

**解决方案**:
- 运行 `npm audit fix` 修复（可能引入破坏性更改）
- 或暂时忽略，不影响开发调试

## 💡 开发建议

### 1. 使用 TypeScript 严格模式
在 `magic-school-frontend/tsconfig.json` 中启用严格模式，提前发现类型错误。

### 2. 使用 ESLint 和 Prettier
统一代码风格，提高代码质量。

### 3. 组件开发规范
- 使用 TypeScript 类型定义 Props
- 组件职责单一，保持简洁
- 使用注释说明复杂逻辑

### 4. Git 提交规范
使用语义化提交信息：
```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构代码
test: 测试相关
chore: 构建/工具相关
```
