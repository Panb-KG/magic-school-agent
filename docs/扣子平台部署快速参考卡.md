# 🚀 扣子平台部署快速参考卡

## ⚡ 必须配置项（5分钟内完成）

### 1. 选择版本
选择最新的 commit 版本

### 2. 域名配置
- 使用默认域名（推荐）

### 3. 数据库配置
- 保留所有数据表（默认）
- achievements, courses, coursewares, exercises, homeworks, students

### 4. 环境变量（⚠️ 必须配置）

在部署界面的"环境变量"中添加以下变量：

```bash
# JWT密钥
JWT_SECRET=your-super-secret-jwt-key-change-in-production

# 阿里云百炼API密钥（⚠️ 注意：使用 DASHSCOPE_ 前缀，不是 COZE_）
DASHSCOPE_API_KEY=sk-f02e3eb3ddff4251a1aaddac91ccb724

# 阿里云百炼API地址（⚠️ 注意：使用 OPENAI_ 前缀，不是 COZE_）
OPENAI_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# 调试模式
DEBUG=False

# 日志级别
LOG_LEVEL=INFO
```

**⚠️ 重要提醒**：
- ✅ 使用 `DASHSCOPE_API_KEY`（标准命名）
- ✅ 使用 `OPENAI_BASE_URL`（标准命名）
- ❌ 不要使用 `COZE_WORKLOAD_IDENTITY_API_KEY`
- ❌ 不要使用 `COZE_INTEGRATION_MODEL_BASE_URL`

### 5. 端口配置
- 在"更多配置"中设置端口为：`8000`

### 6. 开始部署
- 点击"开始部署"按钮
- 等待3-5分钟

## 🐛 常见问题速查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 无法访问 | 端口配置错误 | 检查端口是否为8000 |
| AI不工作 | API Key未配置 | 检查 DASHSCOPE_API_KEY 是否正确配置 |
| AI不工作 | 环境变量名称错误 | 确认不使用 COZE_ 前缀 |
| 部署失败 | 依赖安装问题 | 查看日志，重新部署 |
| WebSocket失败 | 协议错误 | 使用wss://协议 |

## 📝 关键配置点

### 端口：8000 ⚠️
- 部署到Coze必须设置为8000
- 否则无法访问服务

### API Key：必须配置 ⚠️
- `DASHSCOPE_API_KEY`（⚠️ 不要使用 COZE_ 前缀）
- `OPENAI_BASE_URL`（⚠️ 不要使用 COZE_ 前缀）

### JWT密钥：必须修改 ⚠️
- `JWT_SECRET`
- 生产环境必须使用强随机字符串

## 🔄 环境变量变更说明

### 旧命名（不推荐）
```bash
COZE_WORKLOAD_IDENTITY_API_KEY=...
COZE_INTEGRATION_MODEL_BASE_URL=...
```

### 新命名（推荐）
```bash
DASHSCOPE_API_KEY=...
OPENAI_BASE_URL=...
```

**变更原因**：使用标准的环境变量命名规范，避免特定平台前缀，提高兼容性。
