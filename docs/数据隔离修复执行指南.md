### 步骤 2: 修复剩余工具文件

#### 修复模板

每个工具文件需要按以下方式修改：

```python
# 在文件顶部添加导入
from tools.tool_utils_fixed import get_user_context, check_student_access, require_student_access, get_student_name_by_id

# 修改工具函数参数
def add_xxx(
    student_id: int,  # 从 student_name 改为 student_id
    ...
    runtime: ToolRuntime
) -> str:
    """添加XXX
    
    Args:
        student_id: 学生ID  # 修改注释
        ...
        runtime: 工具运行时上下文
    
    Returns:
        操作结果
    """
    # 在函数开头添加权限检查
    error = require_student_access(runtime, student_id)
    if error:
        return error
    
    # 继续原有逻辑
    db = get_session()
    try:
        student_mgr = StudentManager()
        student = student_mgr.get_student_by_id(db, student_id)
        
        if not student:
            return f"未找到ID为{student_id}的学生"
        
        # ... 其余逻辑保持不变
        
        student_name = student.name or "学生"  # 获取学生姓名用于显示
        return f"成功为{student_name}添加XXX"
    except Exception as e:
        return f"操作失败：{str(e)}"
    finally:
        db.close()
```

## 🎯 关键修复点

### 1. 参数变更

| 原参数 | 新参数 | 说明 |
|--------|--------|------|
| `student_name: str` | `student_id: int` | 使用ID而非姓名，避免同名冲突 |

### 2. 权限检查

在每个涉及 student_id 的工具函数开头添加：

```python
error = require_student_access(runtime, student_id)
if error:
    return error
```

### 3. 用户上下文

使用 `get_user_context(runtime)` 获取当前用户信息。

### 4. 学生姓名获取

使用 `get_student_name_by_id(student_id)` 获取学生姓名用于显示。
