用户指出，在扣子平台部署界面的"更多配置"中，端口等参数都是固定不可修改的。这与之前部署文档中的说明不一致，需要进行修正。

## 📋 实际端口配置

根据扣子平台部署界面的实际情况：

### "更多配置"界面中的配置

| 配置项 | 值 | 是否可修改 | 说明 |
|--------|-----|-----------|------|
| 部署类型 | 动态应用 | ❌ 固定 | 应用类型，不可修改 |
| 服务器资源 | CPU: 1 RAM: 2 GiB | ❌ 固定 | 服务器配置，不可修改 |
| 构建指令 | `bash scripts/setup.sh` | ✅ 可修改 | 构建脚本，通常不需要修改 |
| 运行指令 | `bash scripts/http_run.sh -p 5000` | ✅ 可修改 | 启动脚本，通常不需要修改 |
| 端口配置 | `0.0.0.0:80` | ❌ 固定 | 外部访问端口，不可修改 |

### 端口配置详解

#### 1. 应用内部端口：`5000`
- **说明**: FastAPI应用实际监听的端口
- **配置位置**: 运行指令 `bash scripts/http_run.sh -p 5000` 中的 `-p 5000` 参数
- **可修改性**: 可以通过修改运行指令来更改（通常不需要）
- **作用范围**: 应用内部，用户无法直接访问

#### 2. 外部访问端口：`80`
- **说明**: 外部访问应用的端口
- **配置位置**: "更多配置"中的端口配置
- **可修改性**: 固定不可修改
- **作用范围**: 用户通过这个端口访问应用
- **自动转发**: 系统自动将外部80端口转发到内部5000端口

## 🚫 错误的配置说明（已修正）

### ❌ 错误说法（之前的文档）
> "在'更多配置'中设置端口为：8000"

**问题**：
- "更多配置"中的端口配置是固定的，无法修改
- 端口显示为 `0.0.0.0:80`，不是8000
- 用户无法手动修改端口配置

### ❌ 错误说法（之前的文档）
> "部署到Coze必须设置为8000"

**问题**：
- 实际应用内部端口是5000（由运行指令控制）
- 外部访问端口是80（通过反向代理）
- 8000端口并未使用

## 🔄 端口配置流程

### 1. 应用启动
```bash
# 运行指令（在"更多配置"中）
bash scripts/http_run.sh -p 5000

# 执行流程：
# 1. scripts/http_run.sh 被执行
# 2. 传递 -p 5000 参数
# 3. 调用 python src/main.py -m http -p 5000
# 4. FastAPI应用在0.0.0.0:5000上启动
```

### 2. 端口转发
```
用户访问
  ↓
https://域名 (外部端口80)
  ↓
反向代理 (自动转发)
  ↓
0.0.0.0:5000 (内部端口)
  ↓
FastAPI应用
```

### 3. 访问示例
```bash
# ✅ 正确的访问方式
curl https://mq95rx99rp.coze.site/health

# ❌ 错误的访问方式（指定了内部端口）
curl https://mq95rx99rp.coze.site:5000/health

# ❌ 错误的访问方式（指定了不存在的端口）
curl https://mq95rx99rp.coze.site:8000/health
```
