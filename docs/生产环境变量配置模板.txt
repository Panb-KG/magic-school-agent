# ============================================
# 环境变量命名规范说明

# ✅ 推荐的环境变量命名
# DASHSCOPE_API_KEY - 阿里云百炼API密钥（标准命名）

# ❌ 不推荐的环境变量命名
# COZE_WORKLOAD_IDENTITY_API_KEY - 不应以COZE_开头
# COZE_INTEGRATION_MODEL_BASE_URL - 不应以COZE_开头

# 原因：
# 1. 使用标准的环境变量命名规范，提高兼容性
# 2. 便于在不同部署平台之间迁移
# 3. 符合行业最佳实践
# 4. 避免与特定平台的前缀冲突

# ============================================
# 注意事项
# ============================================

# 1. 必须配置的环境变量：
#    - JWT_SECRET
#    - DASHSCOPE_API_KEY
#    - OPENAI_BASE_URL
#    - PORT（部署到Coze时必须设置为8000）

# 2. 安全建议：
#    - 生产环境必须修改 JWT_SECRET
#    - 不要将 API Key 提交到代码仓库
#    - 使用强密码和密钥
#    - 定期更新密钥

# 3. 配置建议：
#    - 生产环境设置 DEBUG=False
#    - 根据服务器配置调整 WORKERS
#    - 合理设置速率限制
#    - 启用日志记录和监控

# 4. 部署到Coze平台时的特殊要求：
#    - PORT 必须设置为 8000
#    - HOST 可以设置为 0.0.0.0
#    - 如果使用Coze提供的数据库，DATABASE_URL 可以不配置

# 5. 环境变量前缀规则：
#    - 避免使用特定平台的前缀（如 COZE_）
#    - 使用通用的、标准化的命名
#    - 优先使用行业通用的环境变量名称
